---#  CodeSpeak는 무슨 언어일까? 코드 대신 스펙을 유지하는 AI 시대의 프로그래밍 언어

![](https://velog.velcdn.com/images/pobi/post/b3ec38ee-86c8-4fd6-9b78-6d682fa132dc/image.png)

요즘 AI 코딩 도구는 많지만, 대부분은 **“코드를 더 빨리 쓰게 해주는 보조 도구”**에 가깝다. 반면 CodeSpeak는 공식적으로 자신을 **LLM 기반 차세대 프로그래밍 언어**라고 소개한다.

핵심은 자동완성의 개선이 아니라, 개발자가 직접 유지해야 하는 대상을 **긴 소스코드**에서 **짧고 읽기 쉬운 스펙(spec)**으로 바꾸겠다는 데 있다.

---

## CodeSpeak는 일반 프로그래밍 언어와 무엇이 다를까?

Python, Java, Rust 같은 언어는 **코드 자체가 소스 오브 트루스(source of truth)**다.
하지만 CodeSpeak에서는 **스펙이 소스 오브 트루스**다.

CodeSpeak 프로젝트의 핵심 파일은 `.cs.md` 확장자를 사용하는 Markdown 기반 스펙 파일이다.

이 파일에는 보통 이런 것들이 들어간다.

* 기능 설명
* 알고리즘 단계
* 예외 처리
* 제약 조건
* 모듈 의존성

예를 들면 이런 식이다.

```
# parse_html

Goal:
HTML 문서를 파싱하고 DOM 구조를 반환한다.

Steps:
1. 입력 HTML을 tokenizer로 분해한다
2. 태그 구조를 기반으로 tree를 생성한다
3. invalid markup을 복구한다
4. DOM object를 반환한다
```

즉 **기호 중심 문법 대신 자연어 기반 스펙**을 중심으로 작성한다.

---

## 개발 흐름도 완전히 다르다

일반적인 개발 흐름은 보통 이렇다.

```
코드 작성 → 컴파일 → 실행
```

하지만 CodeSpeak에서는 흐름이 조금 바뀐다.

```
스펙 수정 → codespeak build → 코드 생성 및 수정
```

개발자가 직접 수정하는 것은 **구현 코드가 아니라 요구사항 스펙**이다.
그리고 실제 코드 변경은 LLM 기반 빌드 과정이 수행한다.

쉽게 말해

> 개발자는 "무엇을 해야 하는지"를 수정하고
> AI는 "어떻게 구현할지"를 수정한다.

---

## 코드 대신 스펙을 유지하는 이유

CodeSpeak가 강조하는 핵심 포인트는 이것이다.

**사람이 유지해야 하는 표면적을 줄인다.**

실제로 CodeSpeak 예시들을 보면 스펙은 코드보다 훨씬 짧다.

예를 들어:

* **14줄 spec → 139줄 Python 코드**
* **spec 변경 +23/-3 → 코드 변경 +221/-25**

즉 사람이 직접 관리하는 텍스트는 훨씬 줄어든다.

개발자가 관리하는 것은 이제

```
코드
```

가 아니라

```
스펙
```

이 되는 것이다.

---

## CodeSpeak는 AI 코드 생성기와도 다르다

많은 사람들은 CodeSpeak를 단순한 **AI 코드 생성기**라고 생각할 수 있다.

하지만 공식 설명에 따르면 CodeSpeak는 **chat 기반 도구가 아니다.**

대신 다음 구조를 가진다.

* CLI 기반 워크플로
* 프로젝트 구조 관리
* 스펙 모듈화
* 코드 변경 자동 생성

또한 기존 프로젝트에서도 사용할 수 있도록

```
codespeak takeover
```

같은 기능을 제공한다.

이 기능은 기존 코드에서 **스펙을 추출해 CodeSpeak 프로젝트로 전환**한다.

즉 완전히 새로 시작하는 도구가 아니라
**기존 코드베이스 위에서도 동작하도록 설계된 시스템**이다.

## 아직은 매우 초기 단계다

현재 CodeSpeak는 **Alpha Preview 단계**다.

설치 과정도 아직은 꽤 실험적이다.

예를 들어

* `codespeak-cli`
* `uv`
* Anthropic API key

등을 요구한다.

또한 공식적으로는 여러 언어를 지원한다고 하지만
현재 기능들은 Python 중심으로 발전하고 있는 상태다.

그래서 지금의 CodeSpeak는 완성된 언어라기보다

**“스펙 중심 AI 소프트웨어 엔지니어링 시스템”**

에 가깝다.

---

## 정리

CodeSpeak는 Python을 대체하는 새로운 언어라기보다
**개발자가 유지하는 대상을 코드에서 스펙으로 바꾸려는 시도**에 가깝다.

개발자의 역할도 조금 바뀐다.

기존 개발자

```
알고리즘 + 구현 작성
```

CodeSpeak 개발자

```
스펙 + 제약 + 구조 정의
```

그리고 구현은 LLM이 담당한다.

AI가 코드를 대신 짜주는 시대를 넘어
**“코드를 직접 작성하지 않아도 되는 개발 모델”**을 실험하는 프로젝트라는 점에서
CodeSpeak는 꽤 흥미로운 방향을 보여주는 도구다.
