---# 1. 스프링부트 환경설정
https://start.spring.io/ 
![](https://velog.velcdn.com/images/pobi/post/2e78dba4-3719-43e9-980b-fef1e2928d59/image.png)

나는 이렇게 설정하였다.
스프링부트 기본설정을 다해주는 사이트이다. 
조상님들은 이걸 하나하나 다 치셨다고한다.

## 주의
자바 버전이 자신이 설치한 자바의 버전과 같은지 확인한다.
높거나 낮으면 실행 자체가 안된다.
높으면 될줄 안 사람들은 제대로 체크하자.

# 2. 실행시키자
![](https://velog.velcdn.com/images/pobi/post/d4582073-479b-49bf-9c6a-d261be351d51/image.png)
이쪽에서 처 public 옆에있는 실행버튼을 누르면 되는데
저게 안떠있는 사람들은 자신의 자바 버전이 맞는지 확인해 보자.
<br>
public static void main을 실행시켜보자 그러고나서 localhost:8080으로 들어간다면 에러메시지가 뜰것이다.
그러면 굉장히 잘된것이다. 우리가 아직 아무것도 만들지 않았기 때문에 당연한 결과이다.

# 3. 구조를 보자
![](https://velog.velcdn.com/images/pobi/post/ada10cb9-80b7-40a0-9e7a-e03ae22fae9f/image.png)
공식 문서를 해석하는 능력이 굉장히 중요하다.

1. 톰캣이라는 웹서버를 내장하고 있다. 톰캣을 띄우면서 스프링부트가 같이 올라온다.

2. welcomepage에 먼저 띄우고 없으면 인덱스 템플릿안에서 찾아서 띄운다.
3. 우리가 지금까지 한거는 그냥 파일을 그대로 던져주는것이다. 동적으로 만들어주는게 템플릿엔진이다.
4. viewresolver가 알아서 찾아서 보여준다.
 {ViewName}+.html 이런식으로 찾는다.
 
# 4. 실행파일을 만들어보자.
1. 만든 폴더에서 ```./gradlew build```이걸 해준다.
2. ```cd build```
3. libs로 들어가면 ~~~.jar파일이 생겨야하는데
4. 실행은 java -jar ~~~
5. 이렇게 하면 실행이 된다. (같은 포트로는 두개가 안된다.)
6. 서버 배포할때는 저 .jar파일만 복붙해주고 java -jar 해주면 된다. 
7. ./gradlew clean이라 하면 build폴더가 사라진다.
