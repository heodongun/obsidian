---# 3. 프로그램 구성
이 섹션에서는 프로그램 코드를 파일로 구성하는 방법에 대해 설명합니다. 논리적으로 관련된 함수와 데이터 구조를 같은 파일로 묶고, 파일 내용의 가시성을 제어하는 것과 같은 좋은 습관들을 강조합니다. 그림 2는 이어질 구성 체계를 보여줍니다.

## 3.1 프로그램 파일
하나 또는 여러 개의 C 프로그램 파일로 구성된 프로그램은 반드시 main() 함수를 포함하는 파일이 하나 있어야 합니다. main() 함수는 프로그램의 실행을 이끄는 역할을 합니다. 프로그램 파일의 예시는 다음과 같습니다.

9절에서 설명한 것처럼, 프로그램이 여러 개의 파일을 포함할 만큼 커지면, 논리적으로 관련된 함수와 데이터 구조를 같은 파일로 묶기 위해 캡슐화 및 정보 은닉 기법을 사용해야 합니다. 프로그램을 다음과 같이 구성하세요.

- 프로그램이 무엇을 하는지 설명하는 README 파일을 만드세요.

- main 함수를 논리적으로 관련된 다른 함수들과 함께 프로그램 파일에 두세요.

- main 함수를 제외한 논리적으로 관련된 함수들을 모듈 파일에 묶으세요.

- 변수와 함수의 정의 및 선언을 캡슐화하기 위해 헤더 파일을 사용하세요.

- 재컴파일을 더 효율적으로 만들기 위해 Makefile(작업 자동화 도구)을 작성하세요.
<br>
## 3.2 README 파일
README 파일은 프로그램이 무엇을 하고 어떻게 구성되어 있는지 설명하는 데 사용해야 합니다. 예를 들어 README 파일에는 다음 내용이 포함될 수 있습니다.
- 모든 조건부 컴파일 플래그와 그 의미

- 머신에 종속적인 파일

- 재사용된 컴포넌트의 경로
<br>
## 3.3 표준 라이브러리
표준 라이브러리는 일반적으로 사용되는 함수들의 모음으로, 하나의 파일로 결합되어 있습니다. 함수 라이브러리의 예로는 입력/출력 함수 그룹으로 이루어진 <stdio.h>와 수학 함수들로 이루어진 <math.h>가 있습니다. 라이브러리 파일을 사용할 때는 프로그램에 필요한 함수만 포함하세요. 여러분은 자신만의 루틴 라이브러리를 만들고 그것들을 헤더 파일로 묶을 수 있습니다.
<br>
## 3.4 헤더 파일
헤더 파일은 논리적으로 관련된 아이디어를 캡슐화하는 데 사용됩니다. 예를 들어 "time.h" 헤더 파일은 시간 처리에 필요한 세 개의 상수, 세 개의 타입, 그리고 일곱 개의 함수를 정의하고 선언합니다. 헤더 파일은 프로그램 파일에서 선택적으로 포함될 수 있으므로, 필요한 함수에만 가시성을 제한할 수 있습니다.

헤더 파일은 컴파일 전에 C 소스 파일에 포함됩니다. <stdio.h>와 같은 일부 헤더 파일은 시스템 전반에 걸쳐 정의되어 있으며, C 프로그램을 사용하여 표준 입력/출력 라이브러리를 포함해야 합니다. 다른 헤더 파일들은 단일 프로그램이나 프로그램 모음 내에서 사용됩니다. 헤더 파일의 예시는 9절에 나와 있습니다.

- 시스템 헤더 파일을 포함하려면 #include <시스템_이름>을 사용하세요.

- 사용자 정의 헤더 파일을 포함하려면 #include "사용자_파일"을 사용하세요.
- 여러 프로그램에서 필요로 하는 데이터 정의, 선언, typedef, enum 등을 헤더 파일에 포함하세요.
- 함수별로 헤더 파일을 구성하세요.
- 별도의 하위 시스템에 대한 선언은 별도의 헤더 파일에 넣으세요.
- 특정 플랫폼에서 변경될 수 있는 선언 세트가 있다면, 해당 선언들을 별도의 헤더 파일에 넣으세요.
- 표준 라이브러리 헤더 파일과 이름이 같은 개인 헤더 파일 이름을 피하세요. 예를 들어 #include <math.h>라고 작성하면 의도한 개인 수학 헤더 파일 대신 표준 라이브러리 수학 헤더 파일이 포함될 수 있습니다.
- 함수나 변수를 정의하거나 선언하는 헤더 파일을 포함하세요. 이렇게 하면 컴파일러가 타입 검사를 수행하고 외부 선언이 정의와 항상 일치하는지 확인할 수 있습니다.
- 헤더 파일을 중첩하지 마세요. 각 프로그램 파일에 필요한 각 헤더 파일을 명시적으로 #include 하세요.
- 헤더 파일의 머리말(prolog)에서 해당 헤더 파일이 제대로 작동하기 위해 포함해야 하는 다른 헤더 파일들을 설명하세요.
<br>
## 3.5 모듈 파일
모듈 파일은 논리적으로 관련된 함수, 상수, 타입 정의 및 선언을 포함합니다. 모듈은 main() 함수를 포함하지 않는다는 점을 제외하고는 프로그램 파일과 유사합니다.
<br>
## 3.6 Makefile
Makefile은 일부 시스템에서 컴파일 단계를 효율적으로 자동화하는 메커니즘을 제공하는 데 사용됩니다. Makefile을 사용하면 마지막 컴파일 이후 변경된 유틸리티 컴파일 파일만 다시 컴파일할 수 있습니다. Makefile을 사용하면 모든 컴파일 명령을 저장하여 긴 명령들을 간단하게 만들 수 있습니다. Makefile의 예시는 9절에 나와 있습니다.

- 프로그램의 일부로 포함될 모든 파일을 나열하세요.
- 어떤 파일들이 라이브러리의 일부인지 설명하는 주석을 포함합니다.
- 암시적 및 명시적 규칙을 사용하여 소스 파일과 관련된 헤더 파일 간의 의존성을 보여줍니다.
<br>
## 3.7 표준 파일 이름 접미사
소스 코드 파일 이름의 권장 형식은 선택적인 접두사(예: 하위 시스템을 나타냄), 기본 이름, 선택적인 마침표, 그리고 접미사입니다. 기본 이름은 고유해야 하며(길이는 컴파일러에 따라 다르지만 8자 이하로 제한하는 것이 좋습니다), 파일 유형을 나타내는 표준 접미사를 포함해야 합니다. 일부 컴파일러와 도구는 파일 이름에 특정 접미사 규칙을 요구합니다. 그림 3은 몇 가지 표준 접미사를 나열합니다. 또는 컴파일러에서 지시하는 접미사를 사용하세요.
