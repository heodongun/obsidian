# JKD와 JRE가 무엇인가요?
![](https://velog.velcdn.com/images/pobi/post/de79637f-dea6-4e8f-b3ec-558bf6ee16b6/image.png)
JDK ⊃ JRE ⊃ JVM
JDK: 자바 애플리케이션을 개발하기 위한 도구와 라이브러리.
JRE: 자바 애플리케이션을 실행하기 위한 환경.
JVM: 자바 바이트코드를 실행하는 가상 머신.
이정도로 이해하면 될것같다.

# 규칙(?)
### 1. 하나의 소스파일에는 최대 1개의 public클래스만 존재해야한다.
### 2. 소스파일명과 public클래스의 이름은 무조건 같아야한다.
### 3. 아우터 클래스$이너 클래스.class 바이트 코드명은 이렇게 정해진다.
아우터는 바깥 이너는 안쪽 클래스임
여기서 바이트코드는 자바의 코드들을 컴퓨터가 이해할수있는 바이트 명령어로 바뀐것이다.
<br><br>
# 이름짓기
### 변수명 int ourClassNum;
이런식으로 다른 글자들은 대문자로 구분하여준다.
### 상수명 final int MY_DATA;
상수명은 대문자로 다쓰기 때문에 구별할수없어서 _를 붙여준다.
<br><br>
# 참조자료형
자바에서는 우리가 잘알고있는 자료형 말고 다른 자료형이 등장한다.
이른바 참조자료형이다. 참조 자료형에는 어레이, 오브젝트, 스트링이있다.
이것은 우리가 아는 포인터의 개념과 비슷하다고 생각하면 된다.

# 쇼트 서킷
쇼트 서킷은 연산을
수행하는 과정에서 결과가 이미 확정됐을 때 나머지 연산 과정을 생략하는 것을 말한다. 예를들어 (5 > 3) || (3 < 2)를 수행하고자 할 때 왼쪽의 (5 > 3)이 true이므로 오른쪽 항의 결과와 상관없이 결과는 항상 true일 것이다. 따라서 이때는 오른쪽 항인 (3 < 2)를 아예 읽지도 않는 것이 바로 쇼트 서킷이다.
논리 연산자로 논리 연산을 수행할 때는 쇼트 서킷이 적용되지만, 비트 연산자로 논리 연산을 수행하면 쇼트 서킷이 적용되지 않는다.
