---# 자바 여행 2차시 - 부모

![](https://velog.velcdn.com/images/pobi/post/8dea2171-a305-4f6d-bef9-d5a837e852d0/image.jpg)

오늘은 자바에서 캡슐화를 하기위한 중요한 단계인 부모자식 상속에대해서 알아보도록 하자.

# 상속

```
class human{
    int age=1;
    String name="1";
}

class student extends human{
    int studentId=1;
}

class worker extends human{
    int workerId=2;
}
public class Main{
    public static void main(String[] args){
        student s=new student();
        worker w=new worker();
        System.out.println(s.age);
        System.out.println(s.name);
        System.out.println(s.studentId);
        System.out.println(w.age);
        System.out.println(w.name);
        System.out.println(w.workerId);
    }
}
```
이 코드를 보면 worker와 student 둘다 human이라는 부모를 가지고있다.
자식은 부모의 메서드나 필드에있는 변수등등을 쓸수있다.

부모의 생성자를 부를려면
```
class A {
    // 부모 클래스 A의 생성자
    public A(String name) {
        System.out.println("부모 클래스 A의 생성자: " + name);
    }
}

class B extends A {
    // 자식 클래스 B의 생성자
    public B() {
        super("허동운");  // 부모 클래스 A의 생성자를 호출하고 문자열 "허동운"을 전달
        System.out.println("자식 클래스 B의 생성자");
    }
}

public class Main {
    public static void main(String[] args) {
        A a = new B();
    }
}

```
이렇게 super이라는 키워드를 사용하여 부모의 생성자를 쓸수있다.
그리고 주의할점은 자식이 여러명의 부모를 거느려서는 안된다.

# 오버라이드
```
class A{
    void print(){
        System.out.println(1);
    }
}

class B extends A{
    void print(){
        System.out.println(2);
    }
}

public class Main{
    public static void main(String[] args){
        B aa=new B();
        aa.print();
    }
}
```
이런식으로 입력을 하게되면 2가 출력되게 된다.
왜냐하면 자식을 객체로 불러올때 부모도 같이 딸려나오기때문에
클래스 A에있는 print라는 메서드를 클래스B에있는 print가 덮어쓴다라는 느낌으로 보면된다.

하지만 정확히 설명하면 덮어쓰기랑은 다르다 클래스 A에 있는 print()가 사라지는게 아닌 남아있기 때문이다.

이걸 왜 쓰는걸까?

정답은 간단하다. 똑같은 기능을하는데 이름을 다르게 하면 그건 그거대로 헷갈린다. 따라서 보기 편하다.
여러객체를 하나의 타입으로 관리 할수있기때문에 유지보수가 쉽고, 재사용성이 증가하고 느슨한결합으로 유지된다라는 장점도 가지고 있다.

## 부모는 부모가 없냐?
아니다 당연히 있다.
Object Class를 참조하고있다. 
우리가 굳이 쓰지 않아도 컴파일러가 자동으로 extends Object를 추가한다.
