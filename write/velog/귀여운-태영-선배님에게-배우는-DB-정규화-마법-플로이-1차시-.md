# 장난은 그만하고 1NF와 2NF를 자세히 알아보자<br><br><br>

## 우리가 정규화를 해야하는 이유
정규화의 기본적인 목표는 테이블 간에 중복되는 데이터가 발생하지 않도록 하는 것이다. 중복된 데이터를 허용하지 않음으로써 데이터의 무결성을 유지할 수 있고, 이로 인해 데이터베이스 관리에 필요한 저장 공간을 축소시키는 효과가 있습니다.

데이터의 중복을 피하기 위해 데이터를 구조화하고, 그 과정에서 테이블을 더 작은 테이블로 분해하는데 이를 구조화하는 정규화 단계가 정의되어 있습니다.

우리는 그중 1NF를 먼저 볼것입니다.


 ![](https://velog.velcdn.com/images/pobi/post/9d9b1247-6263-4461-9de5-46ba88c0b04b/image.png)


## 1NF하는 이유가 무엇인가요?
1. 중복 데이터 제거
1NF는 각 테이블의 모든 컬럼이 원자값을 가지도록 하는 것입니다. 즉, 하나의 컬럼에 여러 값을 가지지 않도록 합니다. 이를 통해 중복 데이터를 제거할수있습니다.

2. 데이터 무결성 유지
중복 데이터를 허용하면 데이터 무결성을 유지하기 어렵습니다. 예를 들어, 동일한 정보가 여러 곳에 저장되어 있는 경우, 한 곳의 데이터만 수정되고 다른 곳의 데이터는 수정되지 않는 상황이 발생할 수 있습니다. 1NF를 통해 이러한 문제를 방지할 수 있습니다.

3. 데이터베이스 크기 감소
중복된 데이터가 제거됨으로써 데이터베이스의 크기가 줄어들고, 저장 공간이 절약됩니다. 이는 데이터베이스 관리 비용을 절감하는 데 도움이 됩니다.

4. 데이터 삽입, 삭제, 갱신의 이상 현상 방지
1정규화를 통해 테이블이 잘 설계되면 데이터 삽입, 삭제, 갱신 시 발생할 수 있는 이상 현상을 방지할 수 있습니다. 예를 들어, 중복 데이터로 인해 발생할 수 있는 불필요한 데이터 수정 작업을 줄일 수 있습니다.

5. 쿼리문의 성능향상
중복 데이터가 제거되고 테이블이 정규화되면 쿼리 성능이 향상될 수 있습니다. 이는 테이블의 구조가 단순해지고, 인덱스의 효율성이 증가하기 때문입니다.

# 2정규화를 해보자<br><br><br>
## 2정규화는 무엇이며 왜 해야하나요?
2정규화란 부분적 함수종속을 없에고 완전함수종속으로 바꾸는것입니다.
하는 이유는

데이터베이스의 2정규화(Second Normal Form, 2NF)는 1정규화(1NF)를 충족한 후, 더 나아가서 부분적 함수 종속을 제거하여 데이터 무결성과 효율성을 더욱 향상시키는 과정입니다. 2정규화를 하는 주요 이유는 다음과 같습니다:

1. 부분적 함수 종속 제거
부분적 함수 종속: 2정규화는 기본 키의 일부분에만 종속된 비-기본 키 속성을 제거합니다. 이는 기본 키가 두 개 이상의 속성으로 구성된 경우에 주로 발생합니다. 부분적 함수 종속을 제거하면, 데이터의 일관성을 유지하고 불필요한 데이터 중복을 방지할 수 있습니다.
2. 데이터 중복 최소화
중복 데이터 감소: 부분적 함수 종속으로 인해 중복되는 데이터를 줄입니다. 중복 데이터는 데이터베이스의 크기를 증가시키고, 데이터 일관성을 유지하기 어렵게 만듭니다.
3. 데이터 무결성 향상
데이터 일관성: 부분적 함수 종속을 제거함으로써, 데이터베이스의 일관성을 더욱 강화할 수 있습니다. 이는 데이터 갱신, 삽입, 삭제 시 이상 현상(Anomalies)을 줄이는 데 도움이 됩니다.
4. 데이터베이스 구조의 효율성 향상
효율적인 데이터베이스 설계: 2정규화를 통해 테이블을 더 효율적으로 설계할 수 있습니다. 이는 데이터베이스의 성능을 향상시키고, 데이터 접근 속도를 높이며, 쿼리를 더 간단하게 만듭니다.

## 부분?완전? 함수종속?
![](https://velog.velcdn.com/images/pobi/post/eecc7d43-35fc-40a0-8931-48918fe26f4c/image.png)
이 그림을 보면 학생이라는 테이블에 기본키가 학번과 과목번호이다.
성적은 학번과 과목번호로 결정되는 한편
하지만 학년과 이름은 학번하나로도 결정될수 있다
기본키중 일부로 결정되는것을 부분함수종속,
기본키 전부로 결정되는것을 완전함수종속이라고 한다.
2정규화 자체가 부분함수종속을 제거하고 완전함수종속으로 만드는 과정이기
때문에 정말 중요한 개념이라고 할수있다.




## 2정규화를 해보자
![](https://velog.velcdn.com/images/pobi/post/b3a20fa1-b0c5-40e0-8aa0-bde8a2165b29/image.png)

우리가 방금배운 완전함수종속과 부분함수종속을 이용하여 무엇이 문제인지 파악해보자.
우선 이 테이블의 기본키는 이름과 학과가 될것이다.
하지만 담당교사와 교실은 완전함수종속이 아닌 부분함수종속이다.
담당교사와 교실은 학과만으로도 정해지기때문이다.
<br><br>

## 그러면 어떻게 해야하나요?
우리는 부분함수종속을 제거할것이다.
즉 모든 테이블에서 완전함수종속 관계가 되게 만들것이다.
따라서 우리는 R(이름,학과)라는 테이블을 만들고
R(학과,담당교사,교실) 이런식으로 테이블을 분리할것이다.

![](https://velog.velcdn.com/images/pobi/post/5f75dcbb-8c02-4fc7-a057-9c418e5dc50e/image.png)

떼어낸 오른쪽 테이블에서는 학과가 담당교사와 교실을 결정을 한다.
이런식으로 모든 관계를 완전함수종속으로 만들어준다.<br><br>
